# 《轻期》产品需求文档（PRD）- V1.0 MVP版

## 一、 产品概述
*   **产品名称**：轻期
*   **产品愿景**：做微信生态内最懂女性、最轻量、最安全、最纯粹的生理期记录工具。
*   **目标用户**：15-45岁，有记录生理期、备孕/避孕需求，且极度反感臃肿App的女性用户。

## 二、 核心功能架构（V1.0 MVP版本）
V1.0 版本做极致极简闭环，摒弃一切干扰功能。

1.  **首页（日历视图）**：无感冷启动、状态展示、日历面板、快捷记录入口。
2.  **记录（数据录入）**：仅记录经期开始日期和结束日期。
3.  **统计（数据可视化）**：周期分析、经期长度趋势。
4.  **我的（个人中心）**：周期设置、提醒设置、**预测原理说明**、数据清除。

## 三、 功能需求明细（按优先级）

### P0级需求（核心基础，不可或缺）

#### 1. 无感冷启动与空状态（0阻力进入）
*   **场景**：用户首次进入小程序。
*   **交互逻辑（渐进式）**：
    *   **进入即看全貌**：无任何信息收集弹窗/表单，直接进入首页日历视图。
    *   **空状态展示**：顶部状态栏显示欢迎语（如：“开启你的纯净记录之旅”），日历无任何预测颜色标识。
    *   **破冰记录**：用户点击日历下方“记一笔”按钮，完成首次“大姨妈来了”的标记，系统即刻后台激活默认算法，渲染预测视图。

#### 2. 首页日历与状态视图
*   **状态卡片**：顶部大字展示当前状态。如：“距离下次经期还有 5 天”、“经期第 2 天”。
    *   *新增巧思：在“距离下次经期还有 X 天”旁边增加一个小小的【ℹ️ 信息图标】，点击可直接唤起“预测原理说明”半屏弹窗。*
*   **智能提示横幅（Smart Banner）**：当用户产生首次记录但未修改过默认设置时，在日历下方出现柔和提示：“当前使用默认28天周期预测。👉去设置真实周期”。用户点击跳转设置，或点击“x”关闭。
*   **日历控件**：
    *   支持月视图左右滑动切换。
    *   **颜色标识**：
        *   红色圆圈：已记录的真实的经期。
        *   浅红色圆圈/虚线：系统预测的未来经期。
        *   紫色/蓝色圆点：预测的排卵日/易孕期。
    *   点击任意日期，下方仅展示该日期是“经期第几天”或“排卵/易孕期”。

#### 3. 极速记录功能（核心亮点）
*   **入口**：首页日历下方悬浮显眼的【记一笔】大按钮（或直接点击具体日期）。
*   **交互**：极简半屏弹窗（Bottom Sheet）或直接在日历面板进行状态切换。
*   **核心记录项**：
    *   **大姨妈来了 / 大姨妈走了**：纯粹的一键状态切换。只记录当次周期的“开始日期”与“结束日期”，无任何额外症状或血量填报项。做到真正的“3秒内完成记录并离开”。

#### 4. 我的 - 周期与基础设置（信息收集后置）
*   **入口**：“我的” Tab页 -> “周期与预测设置”。
*   **字段设置（手动修正预测）**：
    *   **我的经期长度通常是**：默认 5 天（支持滑动选择 2-14 天）。
    *   **两次经期间隔通常是**：默认 28 天（支持滑动选择 20-45 天）。
*   **文案提示**：“连续记录3个月以上，轻期将自动为您测算最准确的平均周期，无需频繁手动修改”。

### P1级需求（完善体验，提升留存）

#### 5. “我们的预测原理” 透明化说明页（信任构建核心）
*   **入口**：
    1. 首页状态栏旁的 【ℹ️】图标。
    2. “我的” Tab 列表中的固定入口【我们的预测原理】。
*   **页面内容（用温暖的大白话向用户解释算法）**：
    *   **“我们如何预测你的经期？”**：说明下一次经期是基于【最近一次真实记录开始日 + 历史平均周期】计算得出。
    *   **“默认值的秘密”**：说明在未了解你之前，系统默认采用28天间隔和5天时长，解释为什么需要用户手动调整或坚持记录。
    *   **“越用越懂你”**：承诺当连续记录满 3 个月后，系统会启动智能算法，根据真实记录加权计算专属周期。
    *   **“关于排卵期的计算”**：科普排卵日倒推14天的基础医学逻辑，并提示此预测仅供参考，不建议作为严格避孕依据。

#### 6. 微信订阅消息提醒
*   **逻辑**：在用户操作时（如进入“我的”页面或开启提醒开关时）引导用户授权微信模板消息。
*   **提醒节点**：
    *   经期来临前 1-2 天提醒（“姨妈快造访啦，记得准备卫生巾哦”）。
    *   排卵日提醒（可选，主要针对备孕模式用户）。

#### 7. 周期统计图表
*   **折线图**：展示近6个月的“经期长度”和“周期长度”波动趋势。
*   **智能平均值计算**：计算用户历史真实的平均周期，系统根据真实数据不断修正预测算法。

### P2级需求（特色功能，亮点打造）

#### 8. 模式切换
*   支持两种模式无缝切换：**常规记录模式**（弱化排卵期提醒）、**备孕模式**（高亮排卵日、易孕期）。

#### 9. UI 主题自定义
*   打破传统生理期App默认粉红色的刻板印象。提供几套极简纯净的配色（如薄荷绿、莫兰迪灰、克莱因蓝），供用户在“我的”页面自由切换。

## 四、 核心算法与数据逻辑说明

1.  **预测算法基础逻辑（默认值策略）**：
    *   **触发条件**：用户首次标记“经期来了”时，若未在设置中填报真实周期，系统采用常量：`Default_Period_Length = 5天`，`Default_Cycle_Length = 28天`。
    *   **排卵日** = 下一次预测月经来潮日往前推 14 天。
    *   **易孕期** = 排卵日的前 5 天 + 后 4 天（共10天）。
    *   **下一次经期预测** = 上一次真实记录的开始日期 + 平均周期长度（默认28天或用户设置值或系统计算值）。
    *   *注：当用户记录了连续 3 个月以上的数据后，系统自动切换为【加权平均法】（近期数据权重更高）来自动调整预测周期。*
2.  **数据纠错**：
    *   如果用户超过 45 天未记录“大姨妈来了”，系统暂停无限期预测，顶部状态变更为“等待大姨妈中”，并温和提示用户是否忘记记录或是否怀孕。

## 五、 交互设计与UI原则

1.  **UI风格**：留白、干净、呼吸感。拒绝花哨的插画，采用扁平化或轻拟物风格的极简图标。
2.  **情感化设计**：文案要有温度。例如不使用冷冰冰的“月经异常”，而是用“最近周期有些波动，注意休息哦”。
3.  **单手操作（极速交互）**：由于砍掉了所有症状记录，【记一笔】可设计为屏幕底部中央的一个醒目大按钮，用户单手拇指即可完成“来/走”的状态切换，真正实现“用完即走”。

## 六、 隐私与合规（极其重要）

1.  **数据纯本地化（完全离线）**：不强制要求微信授权登录，**所有核心数据通过 Storage 保存在用户手机本地缓存中，支持完全离线使用。**
2.  **一键删除与清空**：在设置里提供极为明显的“清除所有数据”按钮，点击后彻底抹除本地所有记录，做到真正的“干净”和“数据主权归属用户”。
3.  **合规声明**：在用户首次进入或查看隐私政策时，必须明确告知：**“本产品所有核心数据仅保存在您的设备本地，绝不上传云端，绝不用于任何第三方广告分析。”**
